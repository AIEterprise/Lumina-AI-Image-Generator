## 2024-10-27 - Converting Interactive `div`s to `<button>`s
**Learning:** In highly customized single-file apps with custom components (like the aspect ratio selector or custom sidebar), it's a common anti-pattern to use `div`s with `onclick` handlers. This completely removes basic keyboard accessibility (tabbing, space/enter activation) and screen reader support out of the box.
**Action:** When inspecting or modifying custom UI widgets (like icon buttons or segmented controls), always ensure the base element is a semantic `<button type="button">`. If making custom toggle/radio groups, ensure `role="radiogroup"`, `role="radio"`, and `aria-checked` are properly implemented and synchronized via JS. Provide `:focus-visible` styles explicitly since custom elements often clear native focus rings.
